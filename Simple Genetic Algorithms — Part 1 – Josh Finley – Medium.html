Simple Genetic Algorithms — Part 1

Genetic algorithms are a type of meta-heuristic that can be used to solve optimization problems. They work by simulating the process of natural selection and evolution.

Recently, I stumbled upon the world of evolutionary computation and more specifically, genetic algorithms (GA). This led me to spend the better part of a couple evenings reading up on the subjects, and eventually to try to create one! I set out with the goal to write a simple GA in Python that will attempt to evolve a given word from random data.

I used the videos for chapter 9 from Daniel Shiffman’s the Nature of Code as a jumping-off point, but only as much as they were absolutely needed. In future parts to this project, I’ll do more research to create a better implementation of my GA.

The code for my project can be found on my GitHub.

1. Components of My GA

Natural selection is a process where the organisms that are the most well-adapted to their environment are given the chance to reproduce and pass on their genetics to another generation. This process is responsible for the variety of living creatures that we see today, and of course, also our own species.

From Shiffman’s videos, I learned that my algorithm would need to go through the following steps until the target word is produced:

  1. Initialization — creating a random initial population
  2. Selection — selecting the fittest members of the population
  3. Reproduction — mixing the “genetics” of two selected members and creating a child.

Also from Shiffman’s videos, I determined that I would need classes for my:

  1. population members (organisms),
  2. and my population.

With all this in mind, I created three files:

  1. organism.py
     This file defines the class for each individual organism. The object contains variables for the organism's genes (its string of characters), its fitness, and its target state. The class also contains functions for creating a new organism with random data and calculating its own fitness with respect to the target string.
  2. population.py This file is by far the longest file in the project, coming in at almost 200 lines. It contains the class for a population of organisms. The object contains variables for the list of organisms, their fitnesses, the percentage of organism's to mutate per population, the target, and a few other things.
  3. The population class also contains essential functions for bootstrapping the initial population, natural selection, mutation, reproduction, and regeneration.
  • main.py
     This is the main file which instantiates the population and then continually regenerates it until the target string is found.

Running the Algorithm

Now for the interesting part: the running of the algorithm!

The string HelloWorld is the first one I attempted to evolve from random characters, and with disappointing results. After a couple of tens of thousands of generations, I had some things that kind of looked like my target, but with only at most half of the characters correct.

After trying a few target strings of different lengths, I realized that my GA has a really hard time with anything over five or six characters. Frustrated, I decided to give it something easy to see if it would work at all: my name, ‘josh’.
